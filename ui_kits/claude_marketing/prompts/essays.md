# 提示词 · 跨境电商思考 · ESSAYS

> 用于通过 DeepSeek API 生成或更新 `data.js → window.ESSAYS` 中的单篇文章。
> 风格：第一人称、克制、运营人能立刻用上。

---

## 输出 schema

```json
{
  "id": "kebab-case-slug-yyyy-mm",
  "week": "第 19 周 · 2026.05",
  "eyebrow": "全大写英文标签 · 2-3 词",
  "title": "≤ 28 字。一个观点，不是一个口号。",
  "deck": "副标题。1-2 句，把 title 的观点展开。≤ 80 字。",
  "paragraphs": [
    "段落 1 ··· 段落 N。每段一个想法。可用 **粗体** 强调（自动转 <strong>）。",
    "数字编号列表写法见示例。"
  ]
}
```

`paragraphs` 是字符串数组，**每个元素渲染为一个 `<p>`**。`**xxx**` 自动转粗体。不要写 `<br>`。

## 写作守则

1. **一篇文章只讲一个判断。** 标题里的观点必须在文章结束时被明确论证或反驳。
2. **必须有"具体到后台"的细节。** 至少一段写"运营今天打开后台，会看见什么变化"。
3. **每篇结尾给运营 2–4 条可执行建议**。用编号列表 + **粗体动词**。
4. **避免空话**："拥抱变化""降本增效""赋能业务"全部禁用。
5. **可以承认不知道**。例：「这个判断我观察了 3 个月，还不够稳，欢迎反驳。」
6. **段落节奏**：6–10 段，单段 80–180 字。第一段用一个具体场景或反常识开场。
7. **结尾**留一个开放钩子，引向下一篇。

## 选题方向（围绕 AI × 跨境电商）

- **运营桌面级**：listing / 客服 / 评价 / 广告 / 选品 / 物流 / 退换货 / 多语种合规
- **能力级**：从"AI 能做什么"翻译成"它能在我们后台具体接管哪一步"
- **组织级**：团队结构、KPI、招聘画像、AI 工具采购决策
- **风险级**：合规、知识产权、客诉、平台政策变化
