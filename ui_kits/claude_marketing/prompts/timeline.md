# 提示词 · AI 技术发展史 · TIMELINE 节点

> 用于通过 DeepSeek API 生成或修订 `data.js → window.TIMELINE` 中的单个节点。
> 单条节点是一棵"AI 演化树"上的一个里程碑。

---

## 输出 schema（严格 JSON · 不要 markdown 包裹）

```json
{
  "year": 2017,
  "era": "Transformer 时代",
  "title": "Transformer 论文发表",
  "subtitle": "Attention Is All You Need · Google",
  "summary": "一句到两句的客观描述：发生了什么、由谁完成、为什么重要。≤80 字。",
  "plain": "用日常语言讲背景与故事。把读者当成完全没接触过 AI 的运营人员。可用 <strong>…</strong> 强调，可换行 <br><br>。120–220 字。",
  "analogy": "用一个生活化的比喻把核心机制讲清楚。可选，但强烈建议有。120–200 字。",
  "mechanism": "技术原理。要准确、不能含糊。允许出现术语，但每个术语首次出现要有一句解释。100–180 字。",
  "impact": "引起的社会反响、后续影响、争议或后续发展。100–180 字。",
  "tag": "理论 | 里程碑 | 突破 | 应用 | 争议 | 商业化"
}
```

字段必须齐全；空字段写 `""`，不要省略 key。

## 风格约束

- **语气**：克制、不煽情、不写"震惊业界""颠覆世界"这类公关话术。给出事实和合理解读，让读者自己得出"哇"的判断。
- **比喻必须可校验**：不要写"像魔法一样"。要写一个具体场景，并能用这个场景类比技术机制。
- **数字要查证**：年份、参数量、价格、得分都必须基于公开可查证的事实；不知道就写"约"或省略，不要编造。
- **不写第一人称**。不写"我们""你"。叙事用第三人称客观语气；只有 `analogy` 中可以使用"想象你…"这类指令体。
- **HTML 标签**：仅允许 `<strong>` 和 `<br>`。不要使用 `<em>`、`<a>`、列表标签。
