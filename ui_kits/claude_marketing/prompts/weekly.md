# 提示词 · 每周 AI 大事件 · WEEKLY

> 用于通过 DeepSeek API 生成或更新 `data.js → window.WEEKLY` 中的整周或单条事件。
> 一周固定 5 条事件，覆盖 5 类。

---

## 输出 schema（整周）

```json
{
  "weekOf": "2026 · 第 19 周 · 5 月 4 日 — 5 月 8 日",
  "updated": "更新于 2026.05.08",
  "next": "下次更新：2026.05.15（每周日晚）",
  "events": [ /* 5 条 event 对象，顺序固定见下 */ ]
}
```

## 输出 schema（单条 event）

```json
{
  "kind": "模型升级 | 开源项目 | 新概念 | 政策 | 公司动态",
  "title": "≤ 22 字。具体的、有数字的、有动词的标题。",
  "teaser": "1 句话摘要，≤ 60 字。和 title 不要重复信息。",
  "glyph": "pulse | branch | spark | shield | box",
  "body": [
    "段首 1：背景一句话 + 三件值得关注的具体变化（用数字、对比、可核实事实）。",
    "1. 第一个变化（具体数据）。",
    "2. 第二个变化（具体数据）。",
    "3. 第三个变化（具体数据）。",
    "段尾：对跨境电商 / 后台运营的直接影响是什么。一句话落地。"
  ]
}
```

`body` 是字符串数组，每个元素渲染成一个段落。

## 5 类事件 → glyph 映射（固定）

| kind     | glyph   | 视觉含义     |
|----------|---------|-------------|
| 模型升级 | pulse   | 跳动条形图  |
| 开源项目 | branch  | 树状分叉    |
| 新概念   | spark   | 思维收束    |
| 政策     | shield  | 盾牌        |
| 公司动态 | box     | 商业容器    |

每周 5 条，**5 个 kind 各一条**，顺序也按上表。

## 风格约束

- **必须有可核实的数字**：版本号、价格、星标数、百分比、日期、参数量。没有数字的"模型升级"不发。
- **不要传播未经证实的传闻**。无信源 = 不写。
- **写"对跨境电商的意义"必须是具体动作**，不要写"将带来巨大影响"。要写"今天起，X 工作可以从 N 步缩短到 M 步"。
- **第一段开头**用一句话点明时间和主体（例："本周一，Anthropic 发布…"）。
- **不允许**："震惊""颠覆""革命""划时代""里程碑"等词。

## 完整正例

```json
{
  "kind": "模型升级",
  "title": "Claude Sonnet 4.5 上线 200K 上下文",
  "teaser": "Anthropic 把 Sonnet 4.5 的上下文窗口扩展到 200K，并降低了 30% 的 API 单价。",
  "glyph": "pulse",
  "body": [
    "本周一，Anthropic 发布 Sonnet 4.5 的更新版本。三个变化值得关注：",
    "1. 上下文从 64K 扩展到 200K——足够一次塞进 30 万字的产品手册或整本商品类目。",
    "2. API 价格下调 30%，输入价格降至 $2.4/M tokens，输出 $12/M tokens。",
    "3. SWE-bench Verified 提升至 73%，工具调用稳定性显著改善。",
    "对跨境电商的直接影响：把整个店铺的全部 listing + 后台数据一次性丢给模型做诊断、改写、合规审查，从工程难题变成几行 prompt 的事。"
  ]
}
```

## 触发指令模板

```
你是 Midea AI Lab 的本周值班编辑。基于以下 5 条原始事件笔记，按 schema 输出本周 WEEKLY JSON：

【原始笔记】
{user_pasted_news_5_lines}

【本周日期】
{week_of}            // 例如：2026.05.04 — 2026.05.08
【本期周次】
{week_label}         // 例如：第 19 周

要求：5 条 events，5 个 kind 各 1 条，按顺序：模型升级 → 开源项目 → 新概念 → 政策 → 公司动态。
无原始素材的 kind，直接告知用户"本周无 X 类事件"，不要编造。
直接输出 JSON，不要解释。
```
